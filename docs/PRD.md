# Drift - 产品需求文档

## 产品概述

**产品名称：** Drift

**产品定位：** 面向创意工作者的轻量级版本管理工具

**产品愿景：** 让创意工作者像程序员管理代码一样管理创意作品

## 目标用户

### 主要用户群体

| 用户类型 | 典型场景 | 核心痛点 | 文件类型 |
|----------|----------|----------|----------|
| **插画师/画画博主** | 多版本创作、客户选稿 | 手动管理文件混乱 | PSD、PNG、AI |
| **平面设计师** | 多方案探索、客户修改 | 版本多了找不到 | PSD、AI、SVG |
| **视频创作者** | 不同剪辑版本 | 大文件管理困难 | MP4、MOV |
| **摄影师** | 不同修图风格 | 同一原图多版本管理 | RAW、JPG |
| **小说作者** | 剧情分支、不同结局 | 版本命名混乱 | TXT、Word、Markdown |
| **剧本创作** | 场景修改、对话调整 | 难以对比修改内容 | TXT、Word、PDF |

### 用户特征

| 特征 | 描述 |
|------|------|
| **技术背景** | 弱，不熟悉Git等工具 |
| **工作方式** | 个人或小团队，非技术协作 |
| **文件类型** | 二进制为主（PSD、图片、视频）+ 文本文件 |
| **核心需求** | 简单保存、快速查找、方便导出 |
| **使用频率** | 日常使用 |

## 用户痛点

现状：手动管理版本

```
文件夹：初稿/
文件夹：修改版1/
文件夹：修改版2/
文件夹：终稿/
文件夹：终稿_最终版/
文件夹：终稿_最终版_改/
文件夹：终稿_最终版_改_客户选这个/
```

**痛点：**
1. 命名混乱，找不到想要的版本
2. 占用大量磁盘空间（重复文件）
3. 无法快速预览版本内容
4. 导出指定版本麻烦
5. 无法尝试不同方向（分支探索）

## 功能需求

### 核心功能（MVP）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **版本保存** | 保存当前状态，支持备注和标签 | P0 |
| **版本列表** | 时间线展示所有版本，支持缩略图预览 | P0 |
| **版本导出** | 导出指定版本到指定位置 | P0 |
| **版本回退** | 恢复到任意历史版本 | P0 |
| **创建分支** | 从任意版本分叉，独立发展 | P0 |
| **暂存功能** | 用户可选择性保存部分文件 | P0 |

### 辅助功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| **版本对比** | 并排查看两个版本差异 | P1 |
| **标签管理** | 自定义标签（初稿、客户版、终稿等） | P1 |
| **项目管理** | 多项目独立管理 | P1 |

### 不做的功能

| 功能 | 原因 |
|------|------|
| **分支合并** | 二进制文件无法自动合并，用户手动选择版本 |
| **分布式** | 面向个人用户，本地即可 |
| **远程仓库** | MVP阶段不做，后续考虑 |

## 用户工作流

### 基本工作流

```bash
# 初始化项目
drift init

# 修改文件后查看状态
drift status

# 添加到暂存区
drift add 章节/第一章.txt
drift add .

# 提交版本
drift save -m "完成前四章"

# 查看历史
drift list

# 导出版本
drift export v1 -o ./交付客户
```

### 分支工作流

```bash
# 创建分支探索新方向
drift branch 尝试方案A
drift switch 尝试方案A

# 在分支上工作
drift add .
drift save -m "方案A初稿"

# 切回主线
drift switch main

# 查看所有分支
drift branch list
```

## 使用场景

### 场景1：画画博主

1. 画师开始画稿，保存版本V1（初稿）
2. 继续修改，保存版本V2（修改版）
3. 客户要求尝试不同风格
4. 创建分支"风格A"，在分支上工作
5. 客户选择V1，导出V1交付

### 场景2：小说作者

1. 作者写完前三章，保存版本V1
2. 继续写第四、五章，保存版本V2
3. 想尝试不同剧情走向
4. 创建分支"结局A"和"结局B"
5. 分别在两个分支上创作
6. 最终选择一个结局，导出完整版本

### 场景3：设计师

1. 设计师完成初稿，保存版本V1
2. 客户要求修改配色
3. 保存版本V2（修改配色）
4. 客户又想看另一个方案
5. 从V1创建分支"方案B"
6. 最终客户选择V2，导出交付
