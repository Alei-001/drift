# Drift - 开发计划

## 开发阶段

### Phase 1：基础框架（2周）

**目标：** 搭建项目骨架，实现基本的对象存储

| 任务 | 描述 | 优先级 |
|------|------|--------|
| 项目结构 | 完善目录结构和模块划分 | P0 |
| Blob存储 | 实现文件内容的哈希存储 | P0 |
| Tree存储 | 实现目录结构的递归存储 | P0 |
| 配置管理 | 读写.drift/config.json | P1 |

**交付物：**
- 可以存储和读取Blob对象
- 可以构建和解析Tree对象
- 基本的配置文件支持

---

### Phase 2：暂存区（2周）

**目标：** 实现文件变更检测和暂存区管理

| 任务 | 描述 | 优先级 |
|------|------|--------|
| 变更检测 | 检测工作区文件变更 | P0 |
| 暂存区管理 | 实现index文件的读写 | P0 |
| drift add | 实现添加文件到暂存区 | P0 |
| drift status | 实现查看工作区状态 | P0 |
| drift reset | 实现清空暂存区 | P1 |

**交付物：**
- 可以检测文件新增、修改、删除
- 可以将文件添加到暂存区
- 可以查看暂存区状态

---

### Phase 3：版本管理（2周）

**目标：** 实现版本提交和历史管理

| 任务 | 描述 | 优先级 |
|------|------|--------|
| Commit对象 | 实现版本记录的创建和存储 | P0 |
| drift save | 实现版本提交 | P0 |
| drift list | 实现版本历史查看 | P0 |
| drift show | 实现版本详情查看 | P1 |
| 版本计数 | 自动递增版本号 | P1 |

**交付物：**
- 可以提交暂存区为新版本
- 可以查看版本历史
- 可以查看版本详情

---

### Phase 4：导出和回退（2周）

**目标：** 实现版本导出和回退功能

| 任务 | 描述 | 优先级 |
|------|------|--------|
| drift export | 实现版本导出 | P0 |
| drift restore | 实现版本回退 | P0 |
| 目录还原 | 递归还原目录结构 | P0 |
| 进度显示 | 大文件处理时显示进度 | P2 |

**交付物：**
- 可以导出指定版本到指定位置
- 可以回退到任意历史版本

---

### Phase 5：分支功能（2周）

**目标：** 实现分支创建和切换

| 任务 | 描述 | 优先级 |
|------|------|--------|
| 分支管理 | 实现分支的创建和存储 | P0 |
| drift branch | 实现分支命令 | P0 |
| drift switch | 实现分支切换 | P0 |
| 分支引用 | 管理分支指向的版本 | P0 |

**交付物：**
- 可以创建分支
- 可以切换分支
- 可以查看所有分支

---

### Phase 6：对比功能（1周）

**目标：** 实现版本对比

| 任务 | 描述 | 优先级 |
|------|------|--------|
| drift diff | 实现版本对比 | P1 |
| 文本diff | 实现文本文件差异显示 | P1 |
| 二进制对比 | 显示二进制文件大小变化 | P2 |

**交付物：**
- 可以对比两个版本的差异
- 文本文件显示具体差异
- 二进制文件显示大小变化

---

### Phase 7：优化和完善（2周）

**目标：** 性能优化和用户体验提升

| 任务 | 描述 | 优先级 |
|------|------|--------|
| 硬链接优化 | 未修改文件使用硬链接 | P1 |
| 大文件支持 | 流式处理大文件 | P1 |
| 错误处理 | 完善错误提示 | P1 |
| 帮助文档 | 完善命令帮助信息 | P2 |
| 忽略规则 | 支持.driftignore文件 | P2 |

**交付物：**
- 性能优化
- 更好的用户体验
- 完善的错误处理

## 时间线

```
Week 1-2:   Phase 1 - 基础框架
Week 3-4:   Phase 2 - 暂存区
Week 5-6:   Phase 3 - 版本管理
Week 7-8:   Phase 4 - 导出和回退
Week 9-10:  Phase 5 - 分支功能
Week 11:    Phase 6 - 对比功能
Week 12-13: Phase 7 - 优化和完善
```

## 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 大文件处理 | 性能问题 | 流式处理、进度显示 |
| 并发安全 | 数据损坏 | 文件锁机制 |
| 跨平台兼容 | 路径问题 | 统一路径处理 |
| 硬链接支持 | 空间浪费 | 检测硬链接支持，降级到复制 |
